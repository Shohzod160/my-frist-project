DNS (Domain Name System) Protocol

  • DNS (Domain Name System) Protocol is a fundamental protocol of the internet that translates human-friendly domain names (e.g., www.google.com) into IP addresses (e.g., 142.250.190.78), which computers use to locate and communicate with each other.
  • How DNS Works:
    • User Request: When a user enters a URL in a browser, the request is sent to a DNS resolver.
    • Recursive Query: The resolver queries different DNS servers to find the IP address of the domain.
    • Response from DNS Servers: The resolver gets the IP address and returns it to the user’s device.
    • Connection to Website: The device uses the IP address to connect to the target server and load the website.
  • Types of DNS Servers:
    • Recursive Resolver: Receives queries from users and finds the IP address by contacting other DNS servers.
    • Root DNS Server: The first step in resolving domain names, directing queries to the correct TLD (Top-Level Domain) servers (e.g., .com, .org).
    • TLD Server: Manages specific domain extensions and points to the appropriate Authoritative DNS Server.
    • Authoritative DNS Server: Stores the actual IP address for the requested domain and returns it to the resolver.
  • DNS Query Types:
    • Recursive Query: The resolver does all the work and returns the final IP address.
    • Iterative Query: The resolver queries multiple servers step by step until it finds the IP.
    • Reverse DNS Lookup: Translates an IP address back into a domain name.
  • DNS Protocol and Ports:
    • Uses UDP (User Datagram Protocol) on Port 53 (for fast, lightweight communication).
    • Uses TCP (Transmission Control Protocol) on Port 53 for large responses (e.g., DNS zone transfers).
    • DNS over HTTPS (DoH) and DNS over TLS (DoT) improve security by encrypting DNS queries.
  • Common DNS Records:
    • A Record: Maps a domain to an IPv4 address.
    • AAAA Record: Maps a domain to an IPv6 address.
    • CNAME Record: Aliases one domain name to another.
    • MX Record: Specifies mail servers for a domain.
    • TXT Record: Stores arbitrary text, often for verification.
  • DNS is essential for internet functionality, ensuring human-friendly web navigation.

DNS (Domen Nomi Tizimi) Protokoli

  • DNS (Domain Name System) Protokoli – bu internetning asosiy protokollaridan biri bo‘lib, odamlar uchun qulay bo‘lgan domen nomlarini (masalan, www.google.com) kompyuterlar ishlatadigan IP-manzillarga (masalan, 142.250.190.78) o‘zgartiradi.
  • DNS Qanday Ishlaydi?
    • Foydalanuvchi So‘rovi: Foydalanuvchi brauzerga URL kiritganda, bu so‘rov DNS rezolveriga yuboriladi.
    • Rekursiv So‘rov: Rezolver IP-manzilni topish uchun turli DNS serverlariga so‘rov yuboradi.
    • DNS Serverlarining Javobi: Rezolver domen nomiga mos keluvchi IP-manzilni topib, uni foydalanuvchi qurilmasiga qaytaradi.
    • Veb-saytga Ulanish: Qurilma ushbu IP-manzildan foydalanib, kerakli serverga ulanadi va veb-saytni yuklaydi.
  • DNS Server Turlari:
    • Rekursiv Rezolver: Foydalanuvchi qurilmasidan so‘rov oladi va IP-manzilni topish uchun boshqa DNS serverlari bilan bog‘lanadi.
    • Root DNS Server: Birinchi bosqich bo‘lib, so‘rovni tegishli TLD (Top-Level Domain – yuqori darajali domen) serveriga yo‘naltiradi (masalan, .com, .org).
    • TLD Server: Muayyan domen kengaytmasini boshqaradi va kerakli Avtorizatsiyalangan DNS serverga yo‘naltiradi.
    • Avtorizatsiyalangan DNS Server: Asl IP-manzilni o‘zida saqlaydi va uni rezolverga qaytaradi.
  • DNS So‘rov Turlari:
    • Rekursiv So‘rov: Rezolver barcha bosqichlarni bajarib, oxirgi IP-manzilni qaytaradi.
    • Iterativ So‘rov: Rezolver bosqichma-bosqich turli serverlardan ma’lumot so‘rab boradi.
    • Teskari DNS Qidiruv: IP-manzilni domen nomiga o‘zgartiradi.
  • DNS Protokoli va Portlari:
    • UDP (User Datagram Protocol) 53-portda ishlaydi (tez va yengil aloqa uchun).
    • TCP (Transmission Control Protocol) 53-portda ishlaydi (katta javoblar, masalan, DNS zonalar almashinuvi uchun).
    • DNS over HTTPS (DoH) va DNS over TLS (DoT) – xavfsizlikni oshirish uchun DNS so‘rovlarini shifrlaydi.
  • DNS Yozuvlari:
    • A Record: Domenni IPv4 manzilga bog‘laydi.
    • AAAA Record: Domenni IPv6 manzilga bog‘laydi.
    • CNAME Record: Bitta domenni boshqa domenga yo‘naltiradi.
    • MX Record: Domen uchun pochta serverlarini ko‘rsatadi.
    • TXT Record: Tekshiruv ma’lumotlari yoki boshqa matnli ma’lumotlarni saqlaydi.

Tarmoq topologiyasi va uning tuzilishi

  • Tarmoq topologiyasi – bu kompyuter tarmog‘idagi qurilmalar va ulanishlarning joylashuvi va tuzilishini ifodalaydi. Bir nechta tarmoq topologiyalari mavjud bo‘lib, har birining o‘ziga xos tuzilishi va xususiyatlari bor:
  • 1. Magistral (Bus) Topologiya
    • Tuzilishi: Barcha qurilmalar bitta markaziy kabelga ("magistral") ulanadi. Ma’lumotlar ikkala yo‘nalishda uzatiladi.
    • Afzalliklari: Kichik tarmoqlar uchun oddiy va arzon.
    • Kamchiliklari: Bitta kabelning ishlamay qolishi butun tarmoqni ishdan chiqarishi mumkin; qurilmalar soni ortishi bilan ishlash sifati pasayadi.
  • 2. Yulduz (Star) Topologiya
    • Tuzilishi: Barcha qurilmalar markaziy tugun (switch yoki hub) bilan bog‘lanadi. Tugun ma’lumot almashinuvini boshqaradi.
    • Afzalliklari: Oson boshqariladi va kengaytiriladi; bitta qurilma ishlamay qolsa, tarmoqqa ta’sir qilmaydi.
    • Kamchiliklari: Agar markaziy tugun ishlamay qolsa, butun tarmoq ishdan chiqadi; ko‘proq kabel talab etiladi.
  • 3. Halqa (Ring) Topologiya
    • Tuzilishi: Qurilmalar yopiq doira shaklida bog‘langan; ma’lumot bir yoki ikkala yo‘nalishda harakatlanishi mumkin.
    • Afzalliklari: Ma’lumotlarni samarali uzatadi; oldindan belgilangan ishlash tartibiga ega.
    • Kamchiliklari: Bitta tugunning ishlamay qolishi butun tarmoqni ishdan chiqarishi mumkin (agar qo‘shaloq halqa ishlatilmasa).
  • 4. To‘r (Mesh) Topologiya
    • Tuzilishi: Har bir qurilma boshqa barcha qurilmalarga ulanadi, to‘liq yoki qisman bog‘langan tarmoq hosil qiladi.
    • Afzalliklari: Juda ishonchli; agar bir ulanish ishlamay qolsa, ma’lumot boshqa yo‘nalish orqali yetkaziladi.
    • Kamchiliklari: Juda qimmat va murakkab, chunki ko‘p ulanish talab etiladi.
  • 5. Daraxt (Tree) yoki Ierarxik Topologiya
    • Tuzilishi: Yulduz va magistral topologiyalarining kombinatsiyasi bo‘lib, ko‘p bosqichli ierarxik tuzilishga ega.
    • Afzalliklari: Katta tarmoqlar uchun mos va oson boshqariladi.
    • Kamchiliklari: Yuqori darajadagi tugun ishlamay qolsa, bir nechta qurilmalar ishdan chiqishi mumkin.
  • 6. Gibrid (Hybrid) Topologiya
    • Tuzilishi: Ikki yoki undan ortiq topologiyalar birlashtirilgan bo‘lib, maxsus ehtiyojlar uchun moslashtirilgan.
    • Afzalliklari: Moslashuvchan va turli vaziyatlar uchun optimallashtirilgan.
    • Kamchiliklari: Dizayni murakkab va narxi yuqori.
