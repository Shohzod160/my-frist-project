Introduction to Web Development

1. Introduction to Web Development and Basic Concepts

Web development: The process of creating websites and web applications.

Basic concepts:

  • HTML (HyperText Markup Language): Structures content.
  • CSS (Cascading Style Sheets): Styles content.
  • JavaScript: Adds interactivity.
  • Web browser: Software to view web pages.
  • Web server: Stores and delivers web pages.
  • URL (Uniform Resource Locator): Web address.

2. Stages of WWW Development. Client-Server Technologies

Stages:

  • Early text-based browsers
  • Graphical browsers
  • Dynamic web pages
  • Web applications
  • The modern web with rich media and interactivity

Client-server:

  • Client (browser): Requests resources.
  • Server: Provides resources (HTML, CSS, JavaScript, images).
  • Communication: Via HTTP/HTTPS.

3. Components of a Web Application. Frontend and Backend Technologies

Components: User interface, application logic, and data storage.

Frontend:

  • What the user sees and interacts with.
  • Technologies: HTML, CSS, JavaScript, frameworks like React, Angular, Vue.js.

Backend:

  • Server-side logic and data management.
  • Technologies: Python (Django, Flask), Node.js (Express), Ruby on Rails, PHP, databases (MySQL, PostgreSQL, MongoDB).

4. What is Hosting?

Hosting is the service of providing storage space and access for websites or web applications on a server connected to the internet. This makes websites accessible to users worldwide.

5. General Structure and Basic Tags, Attributes of an HTML Document

Structure:

  • <!DOCTYPE html>: Document type declaration.
  • <html>: Root element.
  • <head>: Contains metadata (title, links to CSS, etc.).
  • <body>: Contains the visible content.

Basic tags:

  • <h1> to <h6>: Headings.
  • <p>: Paragraph.
  • <div>: Division.
  • <span>: Inline container.

Attributes:

  • href: Links.
  • src: Images.
  • class: CSS styling.
  • id: Unique identifier.
  • style: Inline styling.

6. Working with Images

  • <img>: Embeds images (src, alt, width, height attributes).
  • <map>: Defines an image map (clickable areas).
  • <area>: Defines clickable areas within an image map (shape, coords, href attributes).
  • <picture>: Provides responsive images based on screen size or other criteria (<source> and <img> tags).

7. Working with Tables

  • <table>: Defines a table.
  • <tr>: Table row.
  • <th>: Table header cell.
  • <td>: Table data cell.
  • <caption>: Table caption.
  • Attributes: colspan, rowspan.

8. Working with Links

  • <a>: Defines a hyperlink (href attribute).
  • target attribute: Specifies where to open the linked document (_blank, _self, _parent, _top).
  • Relative and absolute URLs.

9. Working with Lists

  • <ul>: Unordered list.
  • <ol>: Ordered list.
  • <li>: List item.
  • <dl>: Description list.
  • <dt>: Description term.
  • <dd>: Description data.

10. Working with Forms

  • <form>: Defines a form (action, method attributes).
  • <input>: Form input field (type, name, value attributes).
  • <textarea>: Multiline text input.
  • <select>: Drop-down list.
  • <option>: Option in a drop-down list.
  • <button>: Form button.
  • <label>: Label for form elements.

12. Working with Audio, Video, and Graphics in HTML5

14. Introduction to CSS

  • CSS (Cascading Style Sheets): Controls the presentation and layout of HTML documents.
  • Linking methods:
    • Inline CSS: Directly within HTML tags using the style attribute.
    • Internal CSS: Within the <style> tag in the <head> section of an HTML document.
    • External CSS: In a separate .css file, linked using the <link> tag in the <head> section.

15. CSS Syntax. Types of Selectors (Simple, Combined)

  • Syntax: selector { property: value; }
  • Simple selectors:
    • Element selectors (e.g., p, h1).
    • ID selectors (e.g., #myId).
    • Class selectors (e.g., .myClass).
    • Universal selector (*).
  • Combined selectors:
    • Descendant selectors (e.g., div p).
    • Child selectors (e.g., div > p).
    • Adjacent sibling selectors (e.g., h1 + p).
    • General sibling selectors (e.g., h1 ~ p).

16. Types of Selectors (Pseudo-elements, Pseudo-classes, Attribute Selectors)

  • Pseudo-classes: Select elements based on their state (e.g., :hover, :active, :focus, :first-child).
  • Pseudo-elements: Style specific parts of an element (e.g., ::before, ::after, ::first-line, ::first-letter).
  • Attribute selectors: Select elements based on their attributes (e.g., [type="text"], [href^="https"]).

17. Measurement Units

    • in (inches)
  • Relative units:
    • em (relative to the font size of the parent)
    • rem (relative to the root element's font size)

    • vw (viewport width)
    • vh (viewport height)

18. CSS Properties. CSS Page Formatting

19. CSS Color Properties

    • HSLA (hsla(hue, saturation, lightness, alpha))

20. CSS Border Properties

  • border-width: Sets border thickness.
  • border-style: Sets border style (e.g., solid, dashed, dotted).
  • border-color: Sets border color.
  • border: Shorthand property.
  • Individual border properties: border-top, border-right, border-bottom, border-left.
  • Border radius: border-radius.

21. CSS Background Properties

  • background-color: Sets background color.
  • background-image: Sets background image.
  • background-repeat: Controls image repetition.
  • background-position: Sets image position.
  • background-size: Sets image size.
  • background: Shorthand property.

22. CSS Font Properties

  • font-family: Sets font family.
  • font-size: Sets font size.
  • font-weight: Sets font weight (e.g., bold, normal).
  • font-style: Sets font style (e.g., italic).
  • text-decoration: Adds text decorations (e.g., underline).
  • line-height: Sets line height.
  • text-align: Sets text alignment.

23. Margin Properties

  • margin-top, margin-right, margin-bottom, margin-left: Sets margins on individual sides.
  • margin: Shorthand property.
  • Margins create space outside an element's border.

24. Padding Properties

  • padding-top, padding-right, padding-bottom, padding-left: Sets padding on individual sides.
  • padding: Shorthand property.
  • Padding creates space inside an element's border.

25. Box Model Concept. Box Model Rules

  • Box model: Every HTML element is treated as a rectangular box.
  • Components:
    • Content
    • Padding
    • Border
    • Margin
  • Rules:
    • Total element width = content width + padding + border + margin.
    • Total element height = content height + padding + border + margin.
  • box-sizing property can alter the default box model behavior.

30. Working with Functions in JavaScript

  • Functions are blocks of reusable code.
  • Defined using the function keyword, arrow functions, or function expressions.
  • Can accept parameters and return values.
  • Functions are first-class objects in JavaScript.

31. Objects in JavaScript

  • Objects are collections of key-value pairs.
  • Created using object literals ({}) or constructor functions.
  • Properties can be accessed using dot notation or bracket notation.
  • Objects are dynamic and can be modified at runtime.

32. Handling Events Using JavaScript Functions

  • Events are actions or occurrences in the browser (e.g., clicks, mouseovers).
  • Event handlers are functions that respond to events.
  • Events can be added using inline attributes or event listener methods (addEventListener).

33. Regular Expressions in JavaScript

  • Regular expressions are patterns used to match character combinations in strings.
  • Created using the RegExp object or literal notation (/.../).
  • Methods:

    • replace()
    • search()
    • split()

34. Understanding the Document Object Model (DOM)

  • The DOM is a programming interface for HTML and XML documents.
  • Represents the document as a tree structure of objects.
  • Allows JavaScript to access and manipulate the content, structure, and style of web pages.

43. What Are Falsy Values in JS?

  • Falsy values are values that evaluate to false in a boolean context.
  • '' (empty string)
  • 0
  • null
  • undefined
  • NaN (Not-a-Number)
  • false

44. Difference Between == and === Operators

  • == (equality operator): Performs type coercion before comparison.
  • === (strict equality operator): Compares values without type coercion.
  • === is generally preferred to avoid unexpected behavior.

45. Why Is the !! Operator Used?

  • The !! operator is used to coerce a value to a boolean.
  • It effectively converts any value to its corresponding boolean equivalent.

46. What Is the Difference Between call and apply Methods?

  • call() and apply() are used to invoke a function with a specified this value.
  • call() accepts arguments as comma-separated values.
  • apply() accepts arguments as an array.

47. What Is an Event Target (event.target)?

  • event.target refers to the DOM element that triggered the event.

48. How to Check if a Value Is Falsy in JS?

  • Using the ! operator: !value.
  • Using if (!value) { ... } in conditional statements.

49. What Is the Importance of this?

  • The this keyword refers to the context in which a function is executed.
  • Its value depends on how the function is called.
  • In methods, this refers to the object calling the method.
  • In regular functions, this refers to the global object (or undefined in strict mode).

50. What Is the Difference Between var, let, and const Keywords?

  • var: Function-scoped or globally scoped, can be redeclared and reassigned.
  • let: Block-scoped, can be reassigned but not redeclared.
  • const: Block-scoped, cannot be redeclared or reassigned.

51. What Are Default Parameters?

  • Default parameters allow you to specify default values for function parameters.
  • If a parameter is not provided when the function is called, the default value is used.

52. What Is NaN? How to Check if a Value Is NaN?

  • NaN (Not-a-Number) is a special numeric value that represents an undefined or unrepresentable value.
  • Use isNaN() or Number.isNaN() to check if a value is NaN. Number.isNaN() is more reliable.

53. What Is an Object Prototype?

  • A prototype is an object from which other objects inherit properties and methods.
  • Every object in JavaScript has a prototype.
  • Prototype-based inheritance is a core concept in JavaScript.

54. Why Are Functions in JS Called First-Class Objects?

  • Functions can be assigned to variables, passed as arguments to other functions, and returned as values from other functions.
  • They can have properties and methods, just like other objects.

81. MVC Technology

  • MVC (Model-View-Controller) is a software design pattern that separates application logic into three interconnected components:
  • Model: Manages the application's data and business logic.
  • View: Handles the presentation of data to the user.
  • Controller: Acts as an intermediary, handling user input and updating the model and view.
  • This separation improves code organization, maintainability, and reusability.

82. Classes and Objects in PHP

  • PHP supports object-oriented programming (OOP).
  • Classes are blueprints for creating objects, defining properties (data) and methods (functions).
  • Objects are instances of classes, representing specific entities with their own data and behavior.

83. Creating and Using Objects in PHP

  • Classes are defined using the class keyword.
  • Objects are created using the new keyword.
  • Object properties and methods are accessed using the -> operator.

84. Connecting to MySQL in PHP (3 Methods)

  • PHP can connect to MySQL databases using:
  • mysqli: Improved MySQL extension, offering enhanced features and security.
  • PDO (PHP Data Objects): A database abstraction layer, providing a consistent interface for various database systems.
  • mysql: The older MySQL extension, which should not be used in new projects due to security vulnerabilities.

85. CRUD Operations in PHP

  • CRUD stands for Create, Read, Update, and Delete, which are the fundamental operations performed on data in a database.
  • PHP interacts with databases to perform these operations using SQL queries.

86. Creating and Managing MySQL Tables

  • MySQL tables are created and managed using SQL statements:
  • CREATE TABLE: Creates new tables.
  • ALTER TABLE: Modifies existing tables.
  • DROP TABLE: Deletes tables.

87. Creating Databases and Tables in MySQL Using PDO

  • PDO (PHP Data Objects) provides a database-agnostic way to interact with databases.
  • PDO's exec() or query() methods execute SQL statements to create databases and tables.

88. Using Select, Where, Order By, Group By in PHP

  • These are SQL clauses used to retrieve and manipulate data from MySQL databases:
  • SELECT: Retrieves data.
  • WHERE: Filters data.
  • ORDER BY: Sorts data.
  • GROUP BY: Aggregates data.
  • PHP is used to send these SQL queries to the database and process the results.

89. Working with AJAX in PHP

  • AJAX (Asynchronous JavaScript and XML) allows web pages to update content without full page reloads.
  • PHP scripts handle AJAX requests from the client-side, processing data and returning responses (often in JSON format).

90. Mathematical Functions in PHP

  • PHP provides a variety of mathematical functions for calculations:
  • abs(): Absolute value.
  • round(): Rounds a number.
  • ceil(): Rounds up to the nearest integer.
  • floor(): Rounds down to the nearest integer.
  • rand(): Generates a random number.
  • sqrt(): Square root.
  • pow(): Exponentiation.

91. Filters in PHP

  • PHP filters are used for validating and sanitizing user input. This is crucial for security, preventing issues like cross-site scripting (XSS) and SQL injection.
  • filter_var(): Filters a single variable with a specified filter.
  • filter_input(): Gets external variables (like from $_GET, $_POST, $_COOKIE) and filters them.
  • PHP provides various predefined filters for different data types and validation/sanitization needs, such as:
    • FILTER_VALIDATE_EMAIL
    • FILTER_SANITIZE_STRING

92. Callback Functions in PHP

  • A callback function is a function that is passed as an argument to another function.
  • This allows for more flexible and dynamic code.
  • Commonly used with functions like:
    • array_map(): Applies a callback function to each element of an array.
    • array_filter(): Filters elements of an array using a callback function.
    • usort(): Sorts an array using a user-defined comparison function.
  • Callback functions can be named functions or anonymous functions (closures).

93. JSON in PHP

  • JSON (JavaScript Object Notation) is a lightweight data-interchange format. It's widely used for transmitting data between a server and a web application.
  • PHP provides functions for working with JSON:
    • json_encode(): Converts a PHP variable (array, object) into a JSON string.
    • json_decode(): Converts a JSON string into a PHP variable (array, object).

94. Using CMS Technology to Create Websites

  • CMS platforms provide a user-friendly interface for creating and managing website content without requiring extensive coding knowledge.
  • They typically include features for:
    • Creating and editing pages and posts.
    • Managing media files (images, videos).
    • Installing and customizing themes and templates.
    • Adding and managing plugins/extensions for additional functionality.
    • User management.
  • Examples: WordPress, Joomla, Drupal.

95. Advantages and Disadvantages of Using CMS

  • Advantages:
    • Ease of use: Reduces the need for technical skills.
    • Faster development: Pre-built templates and plugins accelerate website creation.
    • Content management: Simplifies content updates and organization.
    • Scalability: Many CMS platforms can handle growing websites.
    • Community support: Large communities provide support and resources.
  • Disadvantages:
    • Customization limitations: May be restricted by the CMS's features.
    • Performance issues: Can be slower than custom-built websites.
    • Security vulnerabilities: Popular CMS platforms are targets for hackers.
    • Plugin/theme compatibility: Conflicts can arise.
    • Learning curve: While easier than coding, some CMS platforms can still be complex.

96. CMS Features

  • Core features:
    • Content creation and editing (WYSIWYG editors).
    • User management and access control.
    • Media library.
    • Template/theme management.
    • Plugin/extension management.
    • SEO tools.
  • Advanced features:
    • E-commerce integration.
    • Multilingual support.
    • Workflow management.
    • Analytics and reporting.
    • API integration.

97. Modern Types of CMS

  • Traditional CMS (Coupled CMS):
    • Tight coupling between the front-end (presentation) and back-end (content management).
    • Examples: WordPress, Drupal, Joomla.
  • Headless CMS (Decoupled CMS):
    • Separates the content repository (back-end) from the presentation layer (front-end).
    • Content is delivered via APIs, allowing for flexible front-end development.
    • Examples: Contentful, Strapi, Sanity.
  • Hybrid CMS:
    • Combines aspects of both traditional and headless CMS, offering flexibility.
    • Allows for both traditional website delivery and API-based content distribution.
  • E-commerce CMS:
    • Specifically designed for online stores.
    • Examples: Shopify, Magento, WooCommerce (WordPress plugin).
  • Cloud-based CMS (SaaS CMS):
    • Hosted and managed by the CMS provider.
    • Examples: Wix, Squarespace.
